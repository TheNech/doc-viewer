# doc-viewer

Веб‑приложение на **Angular 17** для  просмотра многостраничных документов (изображения страниц) с возможность добавления и изменения аннотаций (текст, стили, перетаскивание, размер, отмена действия). Данные документа подгружаются через `HttpClient` из статических моков в `assets/mocks/`.

**Демо:** [https://doc-viewer-mauve.vercel.app/document/1](https://doc-viewer-mauve.vercel.app/document/1)

## Запуск

```bash
npm install
npm start
```

Сборка: `npm run build`. Маршрут документа: `/doc/:id` — для мока `assets/mocks/1.json` используйте id `1`.

## Архитектура (кратко)

| Слой | Назначение |
|------|------------|
| `DocumentApiService` | Загрузка JSON документа, нормализация URL картинок |
| `DocumentViewInteractionService` | Состояние зума, режима размещения, аннотаций, жесты мыши, undo, сохранение в консоль (публичное состояние — `signal` / `computed`) |
| `DocumentViewComponent` | Загрузка по `ActivatedRoute`, провайдер сервиса взаимодействия на экран, глобальные хоткеи |
| `DocumentAnnotationsComponent` | Разметка страниц, превью рисования, список аннотаций; локально — сигнал «активной» страницы по скроллу |
| `DocumentHeaderComponent` | Панель: зум, режим размещения, очистка, undo, «сохранить» |

Сервис взаимодействия **не** `providedIn: 'root'`: создаётся на уровне `DocumentViewComponent`, чтобы состояние не протекало между переходами по маршрутам. При уходе со страницы вызывается `dispose()` — снимаются глобальные слушатели `document` для drag/draw/resize.

## Плюсы текущей реализации

- **Один центр логики** — поведение аннотаций и зума сосредоточено в `DocumentViewInteractionService`, шаблоны остаются тонкими обвязками событий.
- **Нормализованные координаты** (`x`, `y`, `w`, `h` от 0 до 1 относительно страницы) не зависят от текущего масштаба отображения; проще хранить и воспроизводить разметку.
- **Изоляция состояния экрана** — отдельный инстанс сервиса на маршрут документа, явный `dispose()`.
- **Undo с ограничением глубины** (`UNDO_LIMIT`), снимки не копируют лишние ссылки бездумно (стили клонируются при фиксации).
- **Подгонка зума под первую страницу** после загрузки изображения (`onFirstPageLoaded`) улучшает первый показ без ручного зума.
- **Реактивное состояние** — зум, аннотации, выделение, pending, превью рисования и стили в `DocumentViewInteractionService` завязаны на сигналы; UI обновляется предсказуемо без опоры только на Zone.
- **Сигналы в дочернем компоненте** — номер активной страницы в `DocumentAnnotationsComponent` по скроллу/рендеру.

## Минусы и ограничения

- **События мыши вне сигналов** — обработчики `document` для drag/draw/resize по-прежнему императивны; координаты обновляются частыми `mousemove`, что нагружает `annotations.update` при перетаскивании (при необходимости — throttling или возврат к мутации с одним «тиком» сигнала).
- **Частые проходы `afterRender`** — расчёт активной страницы после каждого рендера приложения; при росте UI имеет смысл сузить триггеры (например, только при изменении `zoom`/размера или через `DestroyRef` + отдельный эффект).
- **Сохранение только в консоль** — `saveToConsole` не отправляет данные на сервер; нет версионирования и обработки ошибок сети.
- **Моки вместо API** — `GET assets/mocks/:id.json`; нет авторизации, пагинации, загрузки больших PDF на сервере.
- **Ввод с мыши** — логика ориентирована на `MouseEvent` и `document.mousemove`/`mouseup`; сенсорный ввод и планшеты не учтены.
- **Доступность** — частично (Escape, Ctrl/Cmd+Z); нет полноценной клавиатурной навигации по аннотациям и ролей ARIA для оверлеев.
- **Эвристика «активной страницы»** — скоринг по пересечению с вьюпортом и расстоянию до центра; при нестандартных высотах страниц или встроенном превью возможны скачки номера.

## Известные проблемы и возможные решения

| Проблема | Почему возникает | Направления решения |
|----------|------------------|---------------------|
| После drag/resize клик по фону страницы иногда не фиксирует pending сразу | Используется `ignoreBackgroundCommitUntil` (таймаут ~200 ms), чтобы отличить клик от окончания перетаскивания | Уточнить правила (только `click` vs `pointerup`), перевести на Pointer Events с явным `pointerId`, убрать эвристику времени там, где возможно |
| `afterRender` срабатывает на каждый цикл рендера | Глобальный хук после синхронизации представлений | При проблемах с производительностью — перейти на явные зависимости (`effect` + чтение `zoom`/`document`) или ранний выход, если разметка страниц не менялась |
| Частые `annotations.update` при drag | Иммутабельный `map` на каждый `mousemove` | Throttle движения; или один сигнал‑счётчик «тик геометрии» + мутация в известном объекте (если появится `mutate` в целевой версии Angular) |
| Нет персистентности | Вне рамок текущего прототипа | REST/GraphQL endpoint, оптимистичные сохранения, конфликт‑резолюция по версии документа |
| Один глобальный обработчик клавиш на `document` | Удобно для Escape/undo, но событие на всех страницах приложения | Ограничить `HostListener` условием фокуса/роутера или перенос в директиву с `skip` когда не активен просмотр |
